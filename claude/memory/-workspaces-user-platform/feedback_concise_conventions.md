Convention/guideline additions should be minimal — a single sentence, not multi-paragraph explanations with before/after code examples. Verbose examples consume too much context and are overkill for minor guidelines.

**Why:** User feedback that proposed convention update was "much too verbose, contain too many examples for such a minor guideline, and consume too much context."

**How to apply:** When adding rules to CODING-CONVENTIONS.md, CLAUDE.md, or similar docs, default to a single descriptive sentence. Only add code examples if the rule is genuinely non-obvious from the description alone.
