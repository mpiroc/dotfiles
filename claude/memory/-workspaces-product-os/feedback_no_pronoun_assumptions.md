When referring to a third party (PR reviewer, teammate, customer, anyone whose
pronouns the user has not explicitly shared), don't guess. Default to
they/their or repeat the person's name/handle.

**Why:** The user (Matthew) flagged this directly after I used "his" several
times for a colleague named "terranosaurus" without ever having been told
that person's pronouns. He called it sexist. It's a correction, not a
preference — treat it as a hard rule, not a stylistic nudge.

**How to apply:** When writing about a person whose pronouns are not visible
(GitHub handles, DevRev names, Slack usernames, etc.), use:

- "they / their / them"
- the person's name or handle (repeat instead of pronouning)

If the user or the person themselves later signals different pronouns, switch
to those. Until then, neutral phrasing is the floor.

This applies to all surfaces: written plans, commit messages, PR replies,
in-thread responses, and anything else the user might read.
